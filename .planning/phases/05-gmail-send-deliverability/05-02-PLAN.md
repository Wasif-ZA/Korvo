    - Import `createSupabaseServerClient`, `getOAuth2Client`, `crypto` from Node.js, `NextRequest`, `NextResponse`
    - Authenticate user via `supabase.auth.getUser()` — return 401 if not authenticated
    - Check user is Pro plan via `prisma.profile.findUnique({ where: { userId }, select: { plan: true } })` — return 403 with `{ error: "Pro plan required" }` if not pro
    - Generate `state = crypto.randomBytes(32).toString("hex")`

    - Set httpOnly cookie `gmail_oauth_state` with value `state`, maxAge 600 seconds
    - Return `NextResponse.redirect(authUrl)`

    **app/api/gmail/connect/callback/route.ts** — GET handler (per D-01):
    - Import `getOAuth2Client`, `encryptToken`, `prisma`, `createSupabaseServerClient`
    - Extract `code` and `state` from URL searchParams
    - Validate CSRF: compare `state` against `gmail_oauth_state` cookie — redirect to `/settings?gmail=error` if mismatch (Pitfall 5)
    - Authenticate user via Supabase — redirect to `/settings?gmail=error` if no user
    - Exchange code: `const { tokens } = await oauth2Client.getToken(code)`
    - If `tokens.refresh_token` is null, redirect to `/settings?gmail=error` (Pitfall 1)
    - Upsert GmailToken: `prisma.gmailToken.upsert({ where: { userId: user.id }, create: { userId: user.id, encryptedRefreshToken: encryptToken(tokens.refresh_token!), gmailEmail: null, gmailConnectedAt: new Date(), firstSentAt: null }, update: { encryptedRefreshToken: encryptToken(tokens.refresh_token!), gmailConnectedAt: new Date() } })`
    - Delete the `gmail_oauth_state` cookie
    - Redirect to `/settings?gmail=connected`
    - Wrap in try/catch — on error, redirect to `/settings?gmail=error`

    **app/api/gmail/disconnect/route.ts** — DELETE handler:
    - Authenticate user via Supabase
    - Delete GmailToken: `prisma.gmailToken.delete({ where: { userId: user.id } })`
    - Also clean up Redis keys: delete `gmail:warmup:${userId}:first_sent_at`, `gmail:suspended:${userId}`
    - Return `NextResponse.json({ success: true })`
    - If no token found, return 404

    **app/api/gmail/status/route.ts** — GET handler (per D-10):
    - Authenticate user via Supabase
    - Check `prisma.gmailToken.findUnique({ where: { userId: user.id }, select: { gmailConnectedAt: true, gmailEmail: true } })`

    - Check `gmail:reconnect_required:${userId}` key for token invalidation
    - Return JSON: `{ connected: boolean, gmailEmail: string | null, connectedAt: string | null, dailySent: number, dailyLimit: number, suspended: boolean, reconnectRequired: boolean }`

    **app/api/gmail/send/route.ts** — POST handler (per D-06, D-11):
    - Import `createSupabaseServerClient`, `prisma`, `gmailRedis`, `zod`, `Queue` from bullmq, `QUEUE_NAMES`, `getJitterMs`, `checkAndIncrementDaily`, `checkBounceRate`
    - Validate request body with zod: `z.object({ outreachId: z.string().uuid(), contactId: z.string().uuid() })`
    - Authenticate user via Supabase — return 401 if not authenticated
    - Check user is Pro: `prisma.profile.findUnique({ where: { userId }, select: { plan: true } })` — return 403 if not pro
    - Check Gmail connected: `prisma.gmailToken.findUnique({ where: { userId } })` — return 400 `{ error: "Gmail not connected", code: "GMAIL_NOT_CONNECTED" }` if not
    - Check bounce suspension: `checkBounceRate(userId, gmailRedis)` — return 429 `{ error: "Sending paused — too many bounces. Check your email addresses.", code: "BOUNCE_SUSPENDED" }` if suspended
    - Check daily rate limit: `checkAndIncrementDaily(userId, gmailRedis)` — return 429 `{ error: "You've reached your daily send limit to protect your sender reputation.", code: "DAILY_LIMIT_REACHED", used, limit }` if not allowed
    - Load outreach + contact data: `prisma.outreach.findUnique({ where: { id: outreachId }, include: { contact: true } })` — return 404 if not found
    - Check outreach not already sent: if `outreach.sentAt` is set, return 409 `{ error: "Already sent" }`
    - Load unsubscribe footer from profile or use default: query `prisma.profile.findUnique({ where: { userId }, select: { unsubscribeFooter: true } })`. If null, use default: `"If you'd prefer not to hear from me, just let me know."` (per D-14)
    - Build job data: `{ outreachId, userId, contactId: outreach.contactId, to: outreach.contact.email, subject: outreach.subject, body: outreach.body + "\n\n" + footer }`
    - Enqueue with jitter: `gmailSendQueue.add("send", jobData, { delay: getJitterMs() })` (per D-11)
    - Return 200 `{ queued: true, estimatedDelay: jitterMs }`

    NOTE: The unsubscribe footer field needs to be added to the Profile model. Add `unsubscribeFooter String? @map("unsubscribe_footer")` to the Profile model in schema.prisma. Run `npx prisma db push`.

    **worker/gmail-send.worker.ts** — Replace stub entirely (per D-07, D-14, SEND-06, SEND-07):
    - Import `google` from `googleapis`, `Worker` from `bullmq`, `workerConnection` from `./lib/redis`, `prisma` from `./lib/prisma`, `QUEUE_NAMES`, `GmailSendJobData`, `decryptToken`, `checkAndIncrementDaily`, `markFirstSend`, `recordBounce`, `checkBounceRate`, `getOAuth2Client`
    - Also create a Redis instance in the worker for deliverability counters (import from `./lib/redis` or create inline)
    - Worker processor function:
      1. Extract `{ outreachId, userId, contactId, to, subject, body }` from `job.data` — body already includes unsubscribe footer (appended at enqueue time)
      2. Defense-in-depth rate check: `checkAndIncrementDaily(userId, redis)` — throw if not allowed
      3. Load GmailToken: `prisma.gmailToken.findUnique({ where: { userId } })` — throw `"Gmail not connected"` if not found
      4. Decrypt refresh token: `decryptToken(record.encryptedRefreshToken)`
      5. Create OAuth2Client via `getOAuth2Client()`, call `oauth2Client.setCredentials({ refresh_token: refreshToken })`
      6. Build RFC2822 message: join headers (To, Subject, Content-Type: text/plain; charset=UTF-8, MIME-Version: 1.0) + empty line + body. Base64url encode: `Buffer.from(message).toString("base64").replace(/\+/g, "-").replace(/\//g, "_").replace(/=+$/, "")`
      7. Send: `const gmail = google.gmail({ version: "v1", auth: oauth2Client }); await gmail.users.messages.send({ userId: "me", requestBody: { raw } })`
      8. On success: update `prisma.outreach.update({ where: { id: outreachId }, data: { sentAt: new Date(), sentVia: "gmail_api" } })`
      9. Move contact: `prisma.contact.update({ where: { id: contactId }, data: { pipelineStage: "contacted" } })` (per SEND-06)
      10. Mark first send: `markFirstSend(userId, redis)` — sets warmup day 1 if not set
    - Error handling (per D-03, D-04, Pitfall 2):
      - Catch `invalid_grant` errors: delete GmailToken record, set Redis key `gmail:reconnect_required:${userId}` with value `"true"`, do NOT retry
      - Catch bounce-like errors (message contains "invalid" address indicators): call `recordBounce(userId, to, redis)`, call `checkBounceRate(userId, redis)`, do NOT rethrow (job complete, address bad)
      - All other errors: rethrow for BullMQ retry
    - Worker config: `{ connection: workerConnection, concurrency: 1 }`
    - Keep existing `failed` event listener

    **shared/types/jobs.ts** — No changes needed. The `body` field in `GmailSendJobData` will contain the full body including footer (appended at enqueue time in the send route).
