## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                                  |
| ----- | ----- | ------------------------------------------------------ |
| xs    | 4px   | Icon gaps, dot separators, badge inner padding         |
| sm    | 8px   | Compact element spacing, chip padding, inline gaps     |
| md    | 16px  | Default element spacing, input padding, card inner gap |
| lg    | 24px  | Section padding, card-to-card gap, form field gap      |
| xl    | 32px  | Layout section breaks, major card padding              |
| 2xl   | 48px  | Page vertical rhythm between major sections            |
| 3xl   | 64px  | Page-level spacing (section top/bottom padding)        |

Exceptions:

- Touch targets (buttons, pipeline board drag handles): minimum 44px height — use `h-11` (44px) already enforced in `Button.tsx`
- Kanban column width: 288px (`w-72`) — fixed to ensure readable card wrapping
- Hero input height: 52px (`h-[52px]`) — `heroVariant` on `Input.tsx` for the search form
- Pipeline board horizontal scroll has `pb-10` (40px) for thumb clearance

> Source: `components/ui/Button.tsx` (`h-11`), `components/ui/Input.tsx` (`heroVariant`), `components/app/PipelineBoard.tsx` (`w-72`).

**Accent `#F97316` reserved for these specific elements only:**

- Primary CTA buttons (`variant="primary"` in `Button.tsx`)
- Active/running pipeline step icon (Loader2 spinner)
- Inline text links within contact cards (`hover:text-accent`)
- "New Search →" link in dashboard header
- Italic accent spans (`.italic-accent` — role label in search results H1)
- Copy-to-clipboard button (primary variant)
- Focus ring on inputs and buttons (`focus:ring-accent/20`)
- Hook used badge background tint (`bg-accent-bg`, `text-accent`)

**Accent must NOT be used on:** backgrounds of cards, static labels, nav items, table headers, sidebar backgrounds, stage column headers.

## Interaction Contracts

### Real-Time Pipeline Progress (UI-06, UI-07)

- Supabase Realtime Broadcast channel: `search:{searchId}`
- 4 stages map to step status: `contacts_found` → contacts step complete; `emails_guessed` → emails complete; `research_done` → hooks complete; `drafts_ready` → drafts complete
- On each broadcast event, update the matching step from `running` to `complete` and advance next step to `running`
- Pipeline tracker shown full-screen (centered, `max-w-lg`) replacing search form during run
- On `pipeline_status === "complete"`: navigate to `/search/{jobId}` via `router.push`
- On `pipeline_status === "failed"`: show error toast, return to search form with form state preserved

### Inline Draft Editing (EMAIL-04, UI-05)

- Subject and body are editable inline — no modal
- Subject uses `Input` component; body uses `<textarea>` matching existing `EmailDraft.tsx` pattern
- Edits are debounced 800ms then auto-saved via `PATCH /api/drafts/{id}`
- Unsaved indicator: small `text-text-muted` label appears 200ms after first keystroke
- Copy to Clipboard: copies full formatted string `To: {email}\nSubject: {subject}\n\n{body}`
- Copy success state: button label swaps to `Copied!` with Check icon, reverts after 2000ms
- Mailto link: `<a href="mailto:{email}?subject={encoded}&body={encoded}">Open in Mail</a>` — opens system mail client

### Drag-and-Drop (DASH-02)

- Library: `@dnd-kit/core` (must be installed — not yet in `package.json`)
- Drag handle: visible on hover only (`opacity-0 group-hover:opacity-100`) using `GripVertical` icon from lucide-react
- Drag overlay: semi-transparent clone of `PipelineCard` at `opacity-80`
- Drop target: column background transitions from `bg-surface-alt/30` to `bg-accent-bg/30` on drag-over
- On drop: optimistic UI update (move card immediately), then `PATCH /api/contacts/{id}/stage` — revert on error
- Keyboard/click fallback: `StageSelector` dropdown visible on card menu (`MoreVertical`) for accessibility

### Slide-Over Panel (DASH-03)

- Entry animation: `translate-x-full` to `translate-x-0` in 250ms `ease-out`
- Width: `w-full max-w-[520px]` on desktop; `w-full` on mobile
- Backdrop: `bg-black/20` (lighter than Modal at `bg-black/40` — slide-over is less modal)
- Close: `X` button top-right + ESC key + backdrop click
- Contains: contact name/title/email, ConfidenceBadge, ScoreBreakdown, ResearchCard (Background/Ask This/Mention This), EmailDraft (editable), FollowUpReminder

### Kanban Board Layout

- Horizontal scroll: `overflow-x-auto` with `scrollbar-thin` utility
- Column width: fixed `w-72` (288px) with `flex-shrink-0`
- Mobile: columns scroll horizontally — same layout, no column stacking
- Column count: 6 fixed stages (`identified` → `contacted` → `responded` → `chatted` → `applied` → `interviewing`)
- Max contacts per column displayed: unlimited — columns scroll vertically within their height

### Responsive Breakpoints

| Breakpoint                              | Layout                                                                                                   |
| --------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| < 640px (mobile, incl. iPhone SE 375px) | Single column. Kanban scrolls horizontally. SlideOver is full-screen. Search form is full-width stacked. |
| 640px–1024px (tablet)                   | 2-column grid for stats. Kanban horizontal scroll. Sidebar hidden — history accessible via nav.          |
| > 1024px (desktop)                      | 4-column stats grid. `lg:col-span-3` board + `col-span-1` sidebar. SlideOver panel at `max-w-[520px]`.   |

| Skeleton pulse          | `animate-pulse` on `bg-border/10` blocks                                                                                |

All animation durations are multiples of 50ms. No infinite animations except loading spinners (`animate-spin`) and the pipeline status pulse (`animate-pulse`).
