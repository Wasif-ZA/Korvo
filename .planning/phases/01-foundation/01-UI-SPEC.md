# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Covers: landing page (hero, search bar, how-it-works, pricing section), auth modals (Google OAuth prompt, signup modal overlay), dedicated /pricing page, and settings page basics.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — pure Tailwind CSS 4.2.2 (D-18: no component library, no shadcn/ui) |
| Preset | not applicable |
| Component library | none — hand-built with class-variance-authority + clsx + tailwind-merge |
| Icon library | lucide-react 1.7.0 |
| Font | Geist Sans (Next.js 16 default via `next/font/google`) — fallback: system-ui, sans-serif |

**Theme resolution:** The project references "Notion/Teal aesthetic" and the design must be "warm, friendly, approachable (not dev-tool)." This resolves to **light mode only** for Phase 1. The "dark" association in Notion docs is the aesthetic reference to clean typography and generous whitespace, NOT dark background colors. Surface is warm off-white, not pure white or dark grey. Dark mode is deferred.

---

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, inline gaps between label + icon |
| sm | 8px | Compact element spacing — button icon margin, input internal gap |
| md | 16px | Default element spacing — card padding, form field gap, nav item gap |
| lg | 24px | Section padding — card group spacing, modal padding |
| xl | 32px | Layout gaps — hero content stack, two-column split gutter |
| 2xl | 48px | Major section breaks — between hero / how-it-works / pricing |
| 3xl | 64px | Page-level vertical rhythm — top/bottom of full-width sections |

Exceptions:
- Touch targets (buttons, nav links) minimum 44px height regardless of visual size — meets WCAG 2.5.5 AAA target size on mobile (iPhone SE)
- Modal overlay backdrop: full viewport, no token applies

## Typography

All sizes in pixels. TailwindCSS 4 utility classes noted for executor reference.

| Role | Size | Weight | Line Height | Tailwind Class Reference |
|------|------|--------|-------------|--------------------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base font-normal leading-relaxed` |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium leading-snug` |
| Heading | 24px | 600 (semibold) | 1.3 | `text-2xl font-semibold leading-tight` |
| Display | 48px | 700 (bold) | 1.15 | `text-5xl font-bold leading-none` (desktop) / `text-4xl font-bold` (mobile, ≤375px) |

Font weight palette: exactly 2 declared weights — **400 (regular)** for body text and **600 (semibold)** for all headings, labels, and interactive elements. 700 bold used only for the hero display line (1 instance per page).

Sub-label copy (pricing plan feature lists, footnotes): 13px / weight 400 / line-height 1.4 — `text-[13px] font-normal leading-snug`. This is the only exception to the 4-size rule and is used sparingly.

## Color

**Theme:** Light. Warm off-white surfaces, teal accent, dark slate text.

| Role | Hex | Tailwind Token Reference | Usage |
|------|-----|--------------------------|-------|
| Dominant (60%) | `#FAFAF8` | Custom: `bg-warm-white` / `bg-[#FAFAF8]` | Page background, hero section background |
| Secondary (30%) | `#F4F3F0` | Custom: `bg-warm-muted` / `bg-[#F4F3F0]` | Cards, nav bar, modal backdrop surface, pricing card backgrounds, how-it-works cards |
| Accent (10%) | `#0D9488` | `bg-teal-600` (Tailwind built-in) | Specific reserved elements only — see below |
| Text (primary) | `#1C1C1A` | `text-[#1C1C1A]` | All headings, body text |
| Text (secondary) | `#6B7280` | `text-gray-500` | Captions, meta text, pricing feature list items |
| Destructive | `#DC2626` | `text-red-600` / `bg-red-600` | Destructive actions only — cancel subscription button, error states |
| Border | `#E5E4E0` | `border-[#E5E4E0]` | Card borders, input outlines, dividers |
| Focus ring | `#0D9488` | `ring-teal-600` | Keyboard focus outlines on all interactive elements |

**Accent (#0D9488 teal) is reserved exclusively for:**
1. Primary CTA button fill ("Start for free", "Get started", "Continue with Google")
2. Pricing "Pro" plan card border highlight and "Most popular" badge
3. Active nav link underline indicator
4. Keyboard focus ring on all focusable elements
5. Pricing page: Pro plan CTA button fill
6. Guest search limit modal: primary action button

Accent must NOT be used for: decorative icons, section backgrounds, card fills, hover states on secondary buttons, or any non-interactive element.

**ConfidenceBadge colors** (Phase 4, referenced here for consistency):
- High confidence: `#16A34A` (green-600)
- Medium confidence: `#CA8A04` (yellow-600)
- Low confidence: `#DC2626` (red-600)

## Component Inventory

Hand-built with pure Tailwind. Each component uses `class-variance-authority` for variant management.

### Button

Variants: `primary` | `secondary` | `ghost` | `destructive`

| Variant | Background | Text | Border | Hover |
|---------|-----------|------|--------|-------|
| primary | `#0D9488` | white | none | `#0F766E` (teal-700) |
| secondary | `#F4F3F0` | `#1C1C1A` | `#E5E4E0` | `#E9E8E4` |
| ghost | transparent | `#1C1C1A` | none | `#F4F3F0` bg |
| destructive | `#DC2626` | white | none | `#B91C1C` (red-700) |

Size: height 44px (all variants — enforces touch target). Padding: 16px horizontal. Border radius: 8px (`rounded-lg`).

### Input / Search Bar

Height: 52px on landing hero (prominent), 44px elsewhere. Border radius: 8px. Border: 1px `#E5E4E0`. Focus: 2px ring `#0D9488`. Background: white (`#FFFFFF`). Placeholder text: `text-gray-400`.

Hero search bar is a grouped input row: [Company] [Role] [Location] [Search button] on desktop, stacked 2+2 on tablet, stacked 4 on mobile (≤375px iPhone SE).

### Card

Border radius: 12px (`rounded-xl`). Background: `#F4F3F0`. Border: 1px `#E5E4E0`. Padding: 24px. Shadow: `shadow-sm` (0 1px 2px rgba(0,0,0,0.05)).

### Modal Overlay (Auth Prompt)

This is the Phase 1 guest limit modal (D-03: after 3 guest searches).

- Backdrop: full viewport, `bg-black/40` (semi-transparent, results page visible behind)
- Modal card: centered, max-width 440px, background white (`#FFFFFF`), border radius 16px, padding 32px
- Backdrop click: does NOT dismiss (user must act — results are inaccessible until auth or navigation away)
- Mobile: bottom sheet pattern, slides up from bottom, border radius 16px top corners only

### Nav Bar

Height: 64px. Background: `#FAFAF8`. Border-bottom: 1px `#E5E4E0`. Sticky top. Z-index: 50.
Logo: left. Nav links (center or right depending on auth state). CTA button (rightmost): primary variant, "Get started" (pre-auth) or "Dashboard" (post-auth).

### Pricing Card

Two cards side by side on desktop, stacked on mobile.

Free card: background `#F4F3F0`, border `#E5E4E0`.
Pro card: background white, border 2px `#0D9488`, "Most popular" badge in teal accent.

## Interaction Contract

### Button States

All buttons must implement:
- `default`: defined above per variant
- `hover`: slight darkening (teal-700 for primary, lighter grey for secondary)
- `focus-visible`: 2px offset ring `#0D9488` — never remove focus outline
- `active`: scale(0.97) via `active:scale-[0.97]`
- `disabled`: 50% opacity, cursor-not-allowed, no hover effect

### Form Inputs

- `default`: border `#E5E4E0`
- `focus`: border `#0D9488`, ring 2px `#0D9488` at 50% opacity
- `error`: border `#DC2626`, error message below in `text-red-600 text-sm`
- `disabled`: background `#F4F3F0`, cursor-not-allowed

### Transitions

All interactive state changes: `transition-colors duration-150 ease-in-out`. No transition on layout-affecting properties (avoid CLS). Modal: `transition-opacity duration-200` + scale from 0.95 to 1.0.

### Guest Search Modal Behavior

- Appears after 3rd guest search completes (not during)
- Results page remains visible behind modal (blurred/dimmed but not hidden)
- Backdrop click: no dismiss (intentional friction — user must make a choice)
- "Not now" button: navigates to landing page, guest searches exhausted message shown in nav
- "Continue with Google": triggers Supabase Google OAuth, redirects back to results on success

### Monthly Limit Modal Behavior (signed-in, 5/5 searches used)

- Appears when user attempts a 6th search (not proactively — D-05 says no earlier nudges)
- Can be dismissed with "Remind me next month" — does not reappear until next search attempt
- "Upgrade to Pro" goes to /pricing page, not inline Stripe Checkout (keeps flow simple for Phase 1)

### Annual/Monthly Pricing Toggle

- Toggle state: "Monthly" (default) | "Annually" (shows annual price + "Save 35%" badge)
- Toggle is a pill selector, not a checkbox — clear visual state
- Switching updates both price display and CTA label simultaneously with no page reload
